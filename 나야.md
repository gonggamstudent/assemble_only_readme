# Naya (나야) - Private Messaging App

> 카카오톡처럼 쉽고, Signal처럼 안전하고, 나만의 색으로

## 프로젝트 개요

Naya는 가족과 친구들을 위한 프라이빗 메신저 앱입니다. 카카오톡의 친숙한 UI와 Signal의 강력한 보안(E2EE)을 결합하고, 사용자가 원하는 대로 커스터마이징할 수 있는 메신저를 목표로 합니다.

### 핵심 가치

- **프라이버시 우선**: 개인정보 수집 최소화, 모든 대화 종단간 암호화(E2EE)
- **카톡 개선**: APK 파일 전송, 대용량 파일(2GB), UI 커스터마이징
- **최적화**: 배터리 및 메모리 사용 최소화
- **오픈소스**: 투명한 개발 과정

## 보안 (E2EE)

### Signal Protocol 방식
```
사용자 A                    서버                    사용자 B
────────                   ──────                   ────────
평문: "안녕"
   │
   ▼ (B의 공개키로 암호화)
암호화: "k3j2n4..."
   │
   ├─────────────────▶ 중계만 (읽을 수 없음)
                          │
                          ├─────────────────▶ (B의 개인키로 복호화)
                                              평문: "안녕"
```

**서버가 저장하는 것:**
- ✅ 메타데이터 (누가, 언제, 누구에게)
- ✅ 암호화된 메시지 (읽을 수 없음)
- ❌ 복호화 키 (절대 서버에 전송 안 함)
- ❌ 평문 메시지

**파일도 암호화:**
- APK, 이미지, 동영상 모두 AES-256 암호화
- 암호화 키는 Signal Protocol로 전송
- 서버에는 쓰레기 데이터만 저장

## 성능 최적화

| 항목 | 목표 | 최적화 방법 |
|------|------|-----------|
| **메모리** | 50MB 이하 | 페이지네이션, 청크 처리 |
| **배터리** | 하루 5% 이하 | 백그라운드 WebSocket 종료, FCM 사용 |
| **CPU** | UI 블로킹 0 | Isolate에서 암호화 |
| **네트워크** | 70% 트래픽 감소 | gzip 압축, WebP 이미지 |
| **저장소** | 500MB 이하 | 7일 자동 삭제, 캐시 관리 |

## 개발 환경 설정

### 1. Backend 설정

```bash
# 서버 설정 (Oracle Cloud Free Tier)
# 1. Oracle Cloud 계정 생성
# 2. VM 인스턴스 생성 (Ubuntu 22.04)
# 3. Node.js 18+ 설치
sudo apt update
sudo apt install nodejs npm postgresql

# 프로젝트 클론
git clone https://github.com/yourusername/naya.git
cd naya/backend

# 패키지 설치
npm install

# 환경 변수 설정
cp .env.example .env
# .env 파일 수정

# 데이터베이스 마이그레이션
npm run migrate

# 서버 실행
npm start
```

### 2. Frontend 설정

```bash
cd naya/frontend

# Flutter 패키지 설치
flutter pub get

# 앱 실행 (Android)
flutter run

# 앱 빌드 (APK)
flutter build apk --release
```

## API 엔드포인트

### 인증
```
POST   /api/auth/register      회원가입
POST   /api/auth/login         로그인
POST   /api/auth/refresh       토큰 갱신
```

### 채팅
```
GET    /api/chatrooms          채팅방 목록
POST   /api/chatrooms          채팅방 생성
GET    /api/chatrooms/:id/messages  메시지 조회
```

### 파일
```
POST   /api/files/upload/chunk     청크 업로드
GET    /api/files/:id              파일 다운로드 (Range 지원)
```

### 피드백 & 오류
```
POST   /api/feedback           피드백 전송
POST   /api/error-reports      오류 보고서 전송
```

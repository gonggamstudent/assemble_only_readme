## 개요

MYME Novel은 사용자가 분기형 스토리 구조를 가진 비주얼 노벨(Visual Novel)을 창작하고 배포할 수 있는 **end-to-end 창작 플랫폼**입니다. GPT·DALL-E·Suno 등 최신 AI API를 통합하여 스토리보드 자동 생성, 캐릭터 일러스트 제작, BGM 작곡까지 창작 과정 전반을 AI로 지원합니다.

스토리보드는 노드 기반 플로우 캔버스로 시각화되며, 완성된 작품은 단독 실행 HTML 또는 ZIP 파일로 내보내거나 마켓플레이스에 공개 배포할 수 있습니다.

## 주요 기능

### 🤖 AI 창작 지원
| 기능 | 설명 | 사용 API |
|------|------|----------|
| **스토리보드 자동 생성** | 제목·주인공·컨셉·장르 입력 → 분기형 내러티브 전체 JSON 자동 생성 | GPT-4o-mini |
| **엔딩 무결성 강제** | 해피·노멀·새드·데드 4종 엔딩 포함 여부 검증 후 미비 시 2차 GPT 수정 요청 | GPT-4o-mini |
| **캐릭터 일러스트 생성** | 텍스트 묘사 → 캐릭터 스프라이트 (표정 5종 세트 포함) | DALL-E 3 |
| **배경 이미지 생성** | 장면 설명 → 고품질 배경 일러스트 | DALL-E 3 |
| **BGM 생성** | 장면 분위기·장르 기반 음악 자동 작곡 | Suno API |
| **이미지 편집** | 배경 제거, 인페인팅(표정 변환) | DALL-E 3 / Sharp.js |
| **자동 장면 구성** | 전체 장면에 배경·캐릭터·BGM 병렬 생성 및 자동 배치 | GPT + DALL-E + Suno |
| **스토리 어시스턴트** | 기존 스토리보드 기반 GPT 편집 대화 | GPT-4o-mini |

### 📐 스튜디오 편집기
- **노드 기반 플로우 캔버스** — @xyflow/react로 분기 구조를 인터랙티브하게 시각화
- **페이지 타입 시스템** — dialogue / narration / choice / transition / end (엔딩 4종)
- **선택지 스타일 A/B** — 화면 중앙 오버레이 또는 하단 와이드 버튼 선택
- **연출 효과** — 흔들림 / 페이드 / 플래시 / 블러를 편집기 미리보기에서 실시간 렌더링
- **캐릭터 스테이지 배치** — %-기반 좌표계(posX, posY, scale)로 정밀 포지셔닝

### 🔗 스토리 무결성 시스템
- **플로우 검증** — 고아 페이지, 끊긴 연결, 수렴하지 않는 분기 자동 감지
- **자동 복구** — 잘못된 nextPageId 참조를 분석하여 자동 재연결
- **엔딩 검증** — 4종 엔딩 누락 시 GPT 재요청으로 완성

### 🌐 배포 & 마켓플레이스
- **HTML/ZIP 내보내기** — 오프라인 실행 가능한 단독 HTML 파일 생성 (에셋 번들링 포함)
- **퍼블리싱** — 고유 slug로 공개 배포, 마켓플레이스 등록
- **마켓플레이스** — 장르별 필터, 텍스트 검색, 플레이 카운트 집계

### 자동 장면 구성 파이프라인

AI가 수십 개의 장면에 에셋을 일괄 생성할 때, 순차 처리 시 BGM 폴링(최대 10분/곡)으로 인한 타임아웃 문제를 `Promise.allSettled` 기반 4단계 병렬 파이프라인으로 해결했습니다.

```
Phase 1: 신규 캐릭터 외형 추론 (GPT, 병렬)
    ↓
Phase 2: 캐릭터 이미지 생성 (DALL-E, 병렬)
    ↓
Phase 3: 각 장면별 [배경 이미지 + BGM] 동시 요청 (모든 장면 병렬)
    ↓
Phase 4: 결과 병합 → DB 원자적 저장 (개별 실패 무시, 성공분만 반영)
```

### 에셋 영속성 전략

DALL-E / Suno URL은 1시간 후 만료됩니다. 서버는 생성 즉시 파일을 `/uploads`에 다운로드하여 영구 URL로 교체 저장합니다. ZIP 내보내기 시에는 해당 파일을 번들에 포함해 완전한 오프라인 실행을 보장합니다.

## 데이터 모델 (핵심)

```
Project
├── stages[]
│   ├── pages[]
│   │   ├── pageId, type (dialogue|narration|choice|transition|end)
│   │   ├── content: { text, character, backgroundUrl, ... }
│   │   ├── nextPageId                   ← 단순 진행
│   │   ├── choices[]: { text, nextPageId } ← 분기
│   │   ├── characters[]: { posX, posY, scale, imageUrl }
│   │   ├── effects[], bgmUrl
│   │   ├── endingType (happy|normal|sad|dead)
│   │   └── choiceStyle ('a'|'b')
│   └── endings[]: { title, type, pageId }
└── assets
    ├── characters[]: { name, imageUrl, description, sprites[] }
    ├── backgrounds[]
    └── music[]
```

## 실행 방법

```bash
# 저장소 클론 후 해당 디렉토리로 이동
cd myme-novel

# 의존성 설치
cd server && npm install
cd ../client && npm install

# 환경변수 설정
cp server/.env.example server/.env
# OPENAI_API_KEY, SUNO_API_KEY, MONGODB_URI, JWT_SECRET 등 입력

# Docker로 MongoDB 실행
docker-compose up -d mongodb

# 서버 & 클라이언트 개발 서버 실행
cd server && npm run dev          # :10010
cd client && npm run dev          # :5173
```

## 개발 하이라이트

- **GPT 출력 신뢰성 확보**: AI가 필수 엔딩 4종을 생략하는 문제를 생성 후 검증 + 2차 수정 요청 패턴으로 해결
- **병렬 AI 요청 설계**: 직렬 처리 대비 자동 장면 구성 속도 ~5배 향상, BGM 타임아웃 문제 근본 해결
- **캐릭터 외형 일관성**: 최초 생성 시 `revisedPrompt`를 DB에 저장해 이후 장면에서 동일 묘사로 재생성
- **완전한 오프라인 내보내기**: 에셋 URL 수집 → 로컬 복사 → ZIP 번들링 파이프라인 구현
- **분기 무결성 보장**: 플로우 검증 + 자동 복구로 끊긴 스토리 경로 없이 안정적인 플레이 보장
